# 机器学习课程设计模板（ML-Course-Design-Template）

本仓库作为《机器学习课程设计》统一模板，每个小组都会基于此模板完成自己的项目。

## 一、项目目标

- 选择一个真实或开放数据集；
- 完成数据预处理、特征工程、模型训练与评估；
- 至少对比 **两种不同的模型或方法**；
- 用清晰的可视化和报告说明你的发现与结论。

## 二、可选课题方向（示例，任选其一或自拟题目）

1. **图像分类**  
   - 数据示例：CIFAR-10、MNIST 或自选图像数据集  
   - 模型建议：CNN / ResNet 等

2. **文本情感分析**  
   - 数据示例：电影评论、商品评论（正向/负向）  
   - 模型建议：Logistic Regression + TF-IDF、LSTM、预训练模型

3. **推荐系统**  
   - 场景示例：电影推荐、商品推荐  
   - 模型建议：协同过滤、矩阵分解、浅层神经网络

4. **结构化数据分类/回归**  
   - 数据示例：信用评分、房价预测等  
   - 模型建议：树模型（RF、XGBoost）、线性模型等

5. **时间序列预测**  
   - 数据示例：销售量、电量、流量等时间序列  
   - 模型建议：ARIMA、LSTM/GRU 等

> 若自拟题目，请在 README 顶部写清楚题目名称和数据来源。

## 三、目录结构说明

```text
data/                  # 放数据集（或在 README 说明数据下载方式）
src/
  main.py              # 主程序入口：训练与评估
  utils.py             # 工具函数（如读数据、画图等）
notebooks/
  README.md            # Notebook 使用说明
reports/
  report_template.md   # 报告结构模板
requirements.txt       # Python 依赖列表
```

> 建议开发流程：先在 `notebooks/` 中探索数据 → 把最终方案整理到 `src/main.py` 中 → 在报告中总结。

## 四、运行方式（建议统一）

1. 安装依赖：

   ```bash
   pip install -r requirements.txt
   ```

2. 运行主程序（示例）：

   ```bash
   python src/main.py --model logistic --test-size 0.2
   ```

3. 程序运行完成后，应在终端输出主要指标（如 accuracy），并在 `outputs/` 或 `reports/` 下生成图表或结果文件（如有则自行创建目录）。

## 六、评分参考（示例）

- 完整性（是否包含数据处理、建模、评估）：30%
- 正确性（代码可运行、方法合理）：30%
- 创新性（模型改进、特征工程、分析角度等）：20%
- 报告质量（结构清晰、图表规范）：10%
- 小组协作与个人贡献（commit 记录、组内互评）：10%
